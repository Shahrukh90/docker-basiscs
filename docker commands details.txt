==Container Management==
These commands are used to manage the lifecycle of a container.

docker run: Creates and starts a new container from an image.
docker start <name>: Starts a stopped container.
docker stop <name>: Gracefully stops a running container.
docker kill <name>: Forcefully stops a running container.
docker ps (or docker container ls): Lists all currently running containers.
docker ps --all (or docker container ls -a): Lists all containers, including stopped ones.
docker rm <name>: Removes a stopped container.
docker exec -it <name> <command>: Runs a command inside a running container, often used to open a shell.
docker logs <name>: Fetches and displays the logs of a container.
docker cp: Copies files/folders between a container and the local filesystem.
docker inspect <name>: Displays detailed information on a container or other Docker objects.

==Image Management==
These commands help manage the images that containers are built from.
docker build -t <name> .: Builds a new image from a Dockerfile in the current directory.
docker pull <image>: Downloads an image from a registry (like Docker Hub).
docker push <image>: Uploads an image to a registry.
docker images (or docker image ls): Lists local Docker images.
docker rmi <image>: Removes an image from local storage.
docker search <image>: Searches a Docker registry for images.
docker image prune: Removes all unused or dangling images.
